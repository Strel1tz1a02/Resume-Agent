# 岗位与 JD 分析实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 实现仅粘贴 JD 即可创建岗位、自动生成可编辑结构化分析、保留重复分析历史的本地岗位工作台。

**Architecture:** 后端新增独立的岗位路由和可替换规则服务，`JobPosting.current_jd_analysis_id` 指向当前分析，`JDAnalysis.job_posting_id` 保存历史归属。前端新增岗位 API 客户端和独立 `JobsPage`，首次导入由前端串联“创建岗位”和“创建分析”，其他页面保持现状。

**Tech Stack:** FastAPI、SQLAlchemy 2.x、Pydantic、Alembic、SQLite、pytest、React 18、TypeScript、Vite、Testing Library、Vitest。

## 全局约束

- 本阶段使用简单、确定性的规则服务，不调用真实 LLM，不追求复杂语义识别。
- 创建岗位只要求非空 `raw_jd_text`；人工传入的非空字段优先于规则提取结果。
- 每次重新分析都创建新的 `JDAnalysis`，历史记录不覆盖。
- 仅 `JobPosting` 使用 `current_jd_analysis_id`；`MatchReport.jd_analysis_id` 保持不变。
- 岗位和分析结果都允许人工修正，修改结构化字段不覆盖原始 JD。
- 本阶段不实现匹配、简历生成、投递或导出。
- 后端命令在 Anaconda 环境 `resume-agent` 中运行；前端继续使用现有 pnpm 环境。
- 所有用户界面文案使用中文；新增功能采用 TDD。

    published_at: str | None = None

class JobPostingRead(JobPostingCreate):
