# 投递、Markdown 导出与应用配置实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 跑通 `ResumeVersion -> ApplicationRecord` 的最终 MVP 链路，并提供浏览器控制路径的 Markdown 下载与本地 AppConfig 表单。

**Architecture:** 后端把投递、导出、配置实现为三个独立 FastAPI 路由模块，继续以 `DEFAULT_USER_ID` 隔离数据。前端分别提供独立 API 客户端和页面组件，由现有 `App` 状态导航集成；导出只读取已保存的数据库 Markdown 并返回附件响应。

**Tech Stack:** FastAPI、Pydantic v2、SQLAlchemy 2、SQLite、pytest；React 18、TypeScript、Vite、Vitest、Testing Library、普通 CSS。

## Global Constraints

- 直接在用户明确指定的 `master` 分支实施，不创建 worktree。
- SQLite 中的 `ResumeVersion.markdown_content` 是唯一正式简历版本；后端不保存导出文件。
- 浏览器决定下载路径；API 与 UI 均不得暴露或使用 `default_output_path`。
- 投递只做状态追踪，不自动向招聘网站提交。
- 同一岗位只允许一条 ApplicationRecord，且创建后岗位与简历关联不可修改。
- 状态只允许 `preparing`、`applied`、`written_test`、`interview`、`offer`、`rejected`、`withdrawn`。
- AppConfig 只允许空导出格式列表或 `markdown`，模型配置只保存占位数据。
- 不增加 React Router、TanStack Query、Zustand、UI 框架或新运行时依赖。
- 每个生产代码变更必须先有失败测试，并观察预期失败后再实现。

ILLEGAL_FILENAME_CHARS = re.compile(r'[<>:"/\\|?*\x00-\x1f]')

def sanitize_filename_part(value: str | None, fallback: str) -> str:
    cleaned = ILLEGAL_FILENAME_CHARS.sub("_", (value or "").strip()).strip(" ._")
    return cleaned or fallback

def build_resume_filename(company: str | None, title: str | None, version_id: int) -> str:
    return f"{sanitize_filename_part(company, '未知公司')}_{sanitize_filename_part(title, '未命名岗位')}_简历版本{version_id}.md"

export type ApplicationStatus = "preparing" | "applied" | "written_test" |
  "interview" | "offer" | "rejected" | "withdrawn";

export type ResumeDownload = { blob: Blob; filename: string };

export async function exportResumeVersion(versionId: number): Promise<ResumeDownload> {
  const response = await fetch(`${API_BASE_URL}/resume-versions/${versionId}/export`, { method: "POST" });
  if (!response.ok) throw new Error(`API request failed: ${response.status}`);
  return { blob: await response.blob(), filename: parseDownloadFilename(response.headers.get("Content-Disposition")) };
}
